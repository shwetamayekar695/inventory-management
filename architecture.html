System Architecture

Frontend — Vue 3 (localhost:3000)
  • Vue Router — 6 page views (SPA)
  • useFilters composable — shared filter state
  • api.js — centralised Axios HTTP client
  • Computed properties for derived display data
  • Custom SVG charts · CSS Grid layouts
  • i18n support (en / ja)
HTTP requests with query params ↕ JSON responses
Backend — FastAPI (localhost:8001)
  • REST API — 16 endpoints across 6 resource groups
  • apply_filters() — warehouse / category / status filtering
  • filter_by_month() — month and quarter (Q1–Q4) filtering
  • Pydantic models — request/response validation
  • CORS middleware — allows all origins (dev mode)
Loaded at startup · stays in memory ↑
Data Layer — server/data/*.json
  • 7 JSON files — inventory, orders, demand, backlog, purchase orders, spending, transactions
  • Loaded by mock_data.py into Python lists on server start
  • All filtering applied in-memory at request time
  • Changes do not persist (restart reloads from files)

Request Data Flow

Step 1
User selects filter
useFilters composable updates shared ref (warehouse, category, status, month)
→
Step 2
api.js builds request
Constructs URLSearchParams, skips "all" values, calls Axios GET
→
Step 3
FastAPI filters
apply_filters() + filter_by_month() run on in-memory Python lists
→
Step 4
Pydantic validates
Response serialised against model, returned as JSON
→
Step 5
Vue renders
Raw data stored in ref, computed properties derive display values and chart data

Global Filter System

Time Period
Month (YYYY-MM) or quarter (Q1–Q4 2025). Maps quarters to month arrays.
Warehouse
Exact string match on the warehouse field. Applies to inventory and orders.
Category
Case-insensitive match. Applies to inventory and orders.
Order Status
Processing, Delivered, Backordered, etc. Orders and dashboard only.

Inventory does not support the Time Period filter — it has no time dimension in the data.
